     5→## Project Overview
     6→
     7→AESCConstruct2026 is a SpaceClaim AddIn for ANSYS Discovery (v251). It provides structural modeling tools: frame/profile generation, fasteners, plates, connectors, rib cutouts, engraving, and custom component properties. The UI is WPF (XAML + code-behind) hosted inside SpaceClaim's dockable/floating panels.
     8→
     9→## Build
    10→
    11→- **Solution:** `AESCConstruct2026.sln` (Visual Studio 2017+)
    12→- **Framework:** .NET Framework 4.8, C# 12.0
    13→- **Output:** Class Library DLL
    14→- **Debug output:** `%ProgramData%\SpaceClaim\AddIns\AESCConstruct2026\`
    15→- **Release output:** `Program Files\ANSYS Inc\v251\scdm\Addins\AESCConstruct2026\`
    16→- Build via Visual Studio or `msbuild AESCConstruct2026.sln`
    17→- No test framework is configured; testing is done manually in SpaceClaim
    18→
    19→**AddIn registration:** `AESCConstruct2026.xml` must exist in the output directory alongside the DLL. If missing after build, copy it manually (see README.md).
    20→
    21→## Architecture
    22→
    23→### Entry Point
    24→`Construct2026.cs` implements `IExtensibility` and `IRibbonExtensibility`. The `Connect()` method initializes the SpaceClaim API, validates the license, and registers all ribbon commands with `Command.Create()`. Commands use `KeepAlive(true)` to prevent GC collection.
    25→
    26→### SpaceClaim API
    27→All SpaceClaim types come from `SpaceClaim.Api.V242`. Key aliases used throughout:
    28→- `Application` = `SpaceClaim.Api.V242.Application`
    29→- `Window` = `SpaceClaim.Api.V242.Window`
    30→
    31→### Module Structure
    32→Each feature module follows the pattern: `Module/` (business logic), `UI/` (XAML controls), optionally `Commands/` and `Utilities/`.
    33→
    34→| Module | Purpose |
    35→|--------|---------|
    36→| **FrameGenerator** | Profile extrusion along curves, joint creation. Contains `ProfileBase`/`JointBase` hierarchies with specialized subclasses (Rectangular, H, L, T, U, Circular, DXF, CSV profiles; Miter, Straight, T, Trim joints) |
    37→| **Fastener** | Bolt/Nut/Washer placement from CSV data files |
    38→| **Plates** | Standard plate geometry with holes/slots/fillets |
    39→| **Connector** | TubeLocker-style connectors with cylindrical geometry patterns |
    40→| **RibCutout** | Rib cutout operations on structural members |
    41→| **Engraving** | Text/pattern engraving on faces |
    42→| **CustomComponent** | Custom property assignment to components |
    43→
    44→### UI System
    45→- **UIManager** (`UIMain/UIManager.cs`): Registers docked and floating panel modes for each module's control. Uses `ElementHost` (WinForms-WPF bridge) for SpaceClaim panel integration.
    46→- **Ribbon** (`UIMain/Ribbon.xml`): Defines the "AESC Construct" tab with button groups. Labels resolved via `GetRibbonLabel` callback using the localization system.
    47→- **Localization** (`UIMain/Language.cs`): CSV-based translation loaded from `%ProgramData%\AESCConstruct\Language\languageConstruct.csv`. Access via `Language.Translate("key")`.
    48→
    49→### Licensing
    50→`Licensing/LicenseSpot.cs` wraps LicenseSpot.Framework. Supports node-locked and network licenses. Commands are enabled/disabled based on `ConstructLicenseSpot.IsValid`. License file at `%ProgramData%\IPManager\AESC_License_Construct2026.lic`.
    51→
    52→### Data Files
    53→All runtime data (profile CSVs, fastener CSVs, language files, connector properties) stored under `%ProgramData%\AESCConstruct\`. Paths configured in `Properties/Settings.settings` and `app.config`.
    54→
    55→### Logging
    56→`Utilities/Logger.cs` writes to `%ProgramData%\AESCConstruct\AESCConstruct2026_Log.txt`. Status messages shown in SpaceClaim via `Application.ReportStatus()`.
    57→
    58→## Command Naming Convention
    59→
    60→SpaceClaim commands use dotted names: `AESCConstruct2026.ExportExcel`, `AESCConstruct2026.Plate`, `AESC.Construct.SetMode3D`, etc. These are registered in `Construct2026.cs` and referenced in `Ribbon.xml`.
    61→
    62→## Installer
    63→
    64→Inno Setup script at `Installer/Construct2026Installer.iss`. Outputs to `Installer/Output/`.
    65→
    66→## Key Namespaces
    67→
    68→- `AESCConstruct2026` — root, entry point, resources
    69→- `AESCConstruct2026.FrameGenerator` — profiles, joints, commands, utilities
    70→- `AESCConstruct2026.Fastener` — fastener module and UI
    71→- `AESCConstruct2026.UIMain` — ribbon, panel management, settings, localization
    72→- `AESCConstruct2026.Licensing` — license validation
    73→- `AESCConstruct2026.Utilities` — shared helpers (Logger, DXFImportHelper, CompNameHelper)
    74→

## Key Design Patterns

1. **Master-Occurrence:** Objects have masters (in part space) and occurrences (in assembly/view space)
2. **Interaction Context:** Tools operate in context-space (not scene-space) for proper coordinate handling
3. **Attribute Namespacing:** Use "AddInName.PropertyName" to avoid collisions
4. **Transaction Wrapping:** All modifications use `WriteBlock.ExecuteTask()` for undo
5. **Ribbon/Command Integration:** Commands link UI to code; text/images from localization
6. **Write Block Protection:** Prevents data model changes during interactive operations like dragging

This comprehensive API provides the foundation for the AESCConstruct2026 add-in's structural modeling features across frame generation, fasteners, plates, connectors, rib cutouts, engraving, and custom components.

### 7. COMMON WORKFLOW PATTERNS

**Access the SpaceClaim API:**
```csharp
// From IExtensibility.Connect()
SpaceClaim.Api.V242.Application app = connection.Application;
SpaceClaim.Api.V242.Document doc = app.ActiveDocument;
SpaceClaim.Api.V242.Window win = app.ActiveWindow;
InteractionContext context = win.ActiveContext;
```

**Create geometry:**
```csharp
Part part = Part.Create(doc, "MyPart");
Point pt = Point.Create(0, 0, 0);
Direction dir = Direction.Create(1, 0, 0);
Line line = Line.Create(pt, dir);
ITrimmedCurve segment = CurveSegment.Create(pt, Point.Create(10, 0, 0));
Modeler.Body box = Modeler.Body.CreateBox(Frame.Create(pt, Direction.Create(1,0,0), Direction.Create(0,1,0)), 5, 5, 5);
DesignBody body = DesignBody.Create(part, "Box", box);
```

**Boolean operations:**
```csharp
Modeler.Body target = /* source body */;
Modeler.Body tool = /* tool body */;
target.Unite(new[] { tool }); // Union
target.Subtract(new[] { tool }); // Difference
target.Intersect(new[] { tool }); // Intersection
```
